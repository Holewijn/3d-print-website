# 3D Print Website

Full-stack production app: public website + admin panel + backend API + Postgres,
deployed automatically into a Proxmox LXC container.

## Stack
- **Backend:** Node.js 20, Express, Prisma, PostgreSQL
- **Frontend & Admin:** Next.js 14 (static export), served by Express
- **Process manager:** PM2
- **Payments:** Mollie (iDEAL, Bancontact, Credit Card)
- **Printer integration:** Moonraker REST API
- **Quote engine:** STL volume parser + material/energy/machine cost calculator

## Repo layout
```
backend/        Express API + Prisma + workers
frontend/       Public Next.js site
admin/          Admin panel Next.js (basePath /admin)
prisma/         Schema + seed
scripts/        Proxmox installer + update.sh
ecosystem.config.js   PM2
```

## Deployment (one command on Proxmox host)

1. Push this repo to GitHub: `https://github.com/Holewijn/3d-print-website.git` (branch `main`).
2. SSH to your Proxmox host as `root`.
3. Download and run the installer:

```bash
wget https://raw.githubusercontent.com/Holewijn/3d-print-website/main/scripts/install-proxmox.sh
chmod +x install-proxmox.sh
./install-proxmox.sh
```

The script will:
- Create LXC **CTID 202** (`print3d-app`), Debian 12, 2 cores, 4 GB RAM, 20 GB disk, DHCP on `vmbr0`
- Install Node.js 20, PostgreSQL, PM2, Git
- Clone the repo, install dependencies, run migrations, seed admin user
- Build frontend, admin, and backend
- Start with PM2 and enable startup-on-boot
- Print the admin email/password and container IP

Override defaults via env vars before running, e.g.:
```bash
CTID=210 RAM_MB=8192 DISK_GB=40 NET_CONFIG="name=eth0,bridge=vmbr0,ip=192.168.1.50/24,gw=192.168.1.1" ./install-proxmox.sh
```

## After install

- **Public site:** `http://<container-ip>:3000`
- **Admin panel:** `http://<container-ip>:3000/admin`
- Login with the admin credentials printed at the end of the installer.

### First-time admin tasks
1. **Settings → Mollie** → paste your API key
2. **Settings → Energy Pricing** → set provider + €/kWh
3. **Printers** → add Moonraker URL(s) + €/hour
4. **Inventory** → add filament brands with stock and where-to-buy URL
5. **Pages** → edit Home, Services, About, etc.

## Updating

From the admin dashboard click **"Update from GitHub"**, or SSH into the container:

```bash
pct exec 202 -- bash -lc "/opt/print3d/scripts/update.sh"
```

The update script pulls, installs, migrates, builds, and restarts PM2.
On any failure it automatically rolls back to the previous commit.

### Cronjob (optional)
Inside the container:
```bash
echo "0 4 * * * root /opt/print3d/scripts/update.sh >> /var/log/print3d/update.log 2>&1" > /etc/cron.d/print3d-update
```

## HTTPS / Cloudflare

The container only exposes HTTP on port 3000 (bound to `0.0.0.0`).
TLS is handled externally by Cloudflare — point a Cloudflare Tunnel or proxied
DNS record at `http://<container-ip>:3000`.

## Local development

```bash
pnpm install --recursive
pnpm prisma:generate
pnpm prisma migrate dev
node prisma/seed.js
pnpm --filter ./backend dev   # http://localhost:3000
pnpm --filter ./frontend dev  # http://localhost:3001
pnpm --filter ./admin dev     # http://localhost:3002
```

## Quote engine notes

`backend/src/services/stl.ts` parses both binary and ASCII STL,
computes mesh volume via signed tetrahedra, then estimates:
- weight = volume × density × infill factor
- print time = heuristic based on weight + layer height
- energy = wattage × hours
- total = material + energy + machine + margin %

Tweak `pricing.marginPct` and `pricing.defaultMachineCostHourCents` in admin Settings.

## Credentials & secrets

Generated by the installer and written to `/opt/print3d/.env` (chmod 600):
- `DATABASE_URL` (with random Postgres password)
- `JWT_SECRET` (48 random chars)
- `ADMIN_PASSWORD` (16 random chars, used by `prisma/seed.js` on first run)

The installer prints all credentials at the end. Save them.
